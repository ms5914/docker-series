What is Docker?

Imagine you're a chef who wants to share your secret recipe with restaurants around the world. The challenge? Each kitchen has different equipment, ingredients, and setup. Your perfect dish might turn out completely different in each location.

Docker solves a similar problem in software development. It's a containerization platform that packages your application along with all its dependencies, libraries, and configuration files into a single, portable container. This container can run consistently across any environment - whether it's your laptop, a testing server, or a production cloud platform.

Core Docker Concepts

1. Images

A Docker image is like a blueprint or template for creating containers. It's a read-only package that includes everything needed to run an application: code, runtime, system tools, libraries, and settings. Think of it as a snapshot of a configured system.

# Pull an official image from Docker Hub
docker pull nginx:latest

# List all images on your system
docker images

2. Containers

A container is a running instance of an image. It's the actual execution environment where your application lives and breathes. You can have multiple containers running from the same image, each operating independently.

# Run a container from an image
docker run -d -p 80:80 nginx:latest

# List running containers
docker ps

3. Dockerfile

A Dockerfile is a text file containing step-by-step instructions for building a Docker image. It's like a recipe that tells Docker how to create your custom image.

# Example Dockerfile for a Node.js application
FROM node:16-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]

Getting Started: Your First Container

Let's walk through creating and running your first Docker container. This example will demonstrate the core Docker workflow:

Step 1: Install Docker

Visit docker.com and download Docker Desktop for your operating system. The installation includes the Docker Engine, CLI tools, and a user-friendly desktop interface.

Step 2: Verify Installation

# Check Docker version
docker --version

# Run the hello-world container
docker run hello-world

Step 3: Run a Web Server

# Run an Nginx web server
docker run -d --name my-nginx -p 8080:80 nginx:latest

# Visit http://localhost:8080 in your browser

üîç What Just Happened?

-d: Runs the container in detached mode (in the background)
--name: Assigns a custom name to the container
-p 8080:80: Maps port 8080 on your host to port 80 in the container
